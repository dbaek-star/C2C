# 서브에이전트 공통 규칙

## 호출 방식

모든 Claude-B 호출은 Agent tool을 사용하여 수행한다.

### 사용법

```
Agent({
  description: "Round {N} - {역할 설명}",
  subagent_type: "general-purpose",
  model: "{사용자 선택 모델}",
  prompt: "{동적 생성 프롬프트}"
})
```

### 파라미터

| 파라미터 | 필수 | 설명 |
|---------|------|------|
| `description` | O | 라운드 번호와 역할을 포함한 짧은 설명 (예: "Round 1 - 초안 검토") |
| `prompt` | O | 지시사항 + 컨텍스트 (Claude-A가 매 호출 시 동적 생성) |
| `model` | X | `opus` / `sonnet` / `haiku` (기본: 사용자 선택값) |
| `subagent_type` | X | `"general-purpose"` (기본값 사용) |

### prompt 구조

Claude-A는 매 호출 시 다음 구조로 prompt를 동적 생성한다. **고정 프롬프트 사용 금지** — 주제·라운드·모드·이전 피드백에 따라 자율 구성할 것.

```markdown
## 시스템 지시사항

당신은 다음 도구를 사용할 수 있습니다: Read (파일 읽기), WebSearch (웹 검색), Grep (코드 검색), Glob (파일 검색).
필요 시 적극 활용하되, 파일 수정(Edit/Write)은 하지 마십시오.

{페르소나 지시 — 모드에 따라 다름}
{웹검색 지시 — 주제 유형에 따라 강제/권장/생략}
{모드별 톤 지시}

---

{이전 라운드 컨텍스트 — 라운드에 따라 다름}

---

## 현재 입력 자료

{초안 또는 수정본}
```

**라운드별 컨텍스트 전달:**

| 라운드 | 컨텍스트 전달 방식 |
|--------|-------------------|
| Round 1 | 초안 전문을 prompt에 포함 |
| Round 2 | 초안 전문 + Round 1 리뷰 전문을 prompt에 포함 |
| Round 3+ | Claude-A가 작성한 요약만 포함 + 이전 라운드 파일 경로 제공하여 Read tool 참조 유도 |

### 응답 처리

서브에이전트의 응답은 Agent tool의 반환값(텍스트)으로 수신된다.

Claude-A의 후처리 절차:

1. **응답 수신** — Agent tool 반환값을 확인
2. **파일 저장** — 응답 텍스트를 해당 라운드 파일로 저장 (예: `round1_2_claude-b_review.md`)
3. **웹검색 여부 확인** — 응답 내 "Sources:" 섹션 존재 여부로 판단, `collab_summary.md`에 기록
4. **분석 및 판단** — 응답 내용을 분석하여 판단/수정/결정 수행

### 장애 처리

1. Agent 호출 실패 → 동일 파라미터로 **1회 재시도**
2. 재시도 실패 → Claude-A **단독 수행**, `collab_summary.md`에 서브에이전트 호출 실패 기록
3. 단독 수행 시에도 동일한 산출물 구조(파일명, 폴더 구조) 유지
