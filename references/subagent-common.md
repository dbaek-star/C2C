# 서브에이전트 공통 규칙

## 호출

Agent tool로 Claude-B를 호출한다. `subagent_type: "general-purpose"`, `model`은 사용자 선택값.

서브에이전트 prompt에 반드시 포함할 도구 안내:
```
당신은 다음 도구를 사용할 수 있습니다: Read, WebSearch, Grep, Glob.
필요 시 적극 활용하되, 파일 수정(Edit/Write)은 하지 마십시오.
```

## prompt 구성

고정 프롬프트 사용 금지. 매 호출 시 다음 요소를 조합하여 동적 생성:

1. **도구 안내** (위 참조)
2. **페르소나** (SKILL.md "독립성 보장" 참조)
3. **웹검색 지시** (SKILL.md "프롬프트 생성 규칙" 참조)
4. **모드별 톤** (modes.md "프롬프트 톤" 참조)
5. **이전 라운드 컨텍스트** (아래 참조)
6. **현재 입력 자료** (초안 또는 수정본)

### 컨텍스트 전달

| 라운드 | 방식 |
|--------|------|
| Round 1 | 초안 전문을 prompt에 포함 |
| Round 2 | 초안 + Round 1 리뷰 전문 포함 |
| Round 3+ | Claude-A 작성 요약만 포함 + 파일 경로 제공하여 Read tool 참조 유도 |

Round 3+ 요약 형식:
```
## 이전 라운드 요약
- 핵심 쟁점: ...
- 합의된 사항: ...
- 미합의 사항: ...

참고 파일: {파일 경로 목록}
```

## 응답 처리

1. Agent tool 반환값(텍스트) 수신
2. 해당 라운드 파일로 저장 (예: `round1_2_claude-b_review.md`)
3. 웹검색 여부: 응답 내 "Sources:" 존재 여부로 판단

## 장애 처리

1. Agent 호출 실패 → 동일 파라미터로 **1회 재시도**
2. 재시도 실패 → Claude-A **단독 수행**, `collab_summary.md`에 기록
3. 단독 수행 시에도 동일한 산출물 구조 유지
